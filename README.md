# Bank2Sheets — Сбор банковских операций в Google Sheets.
Система автоматически собирает транзакции юрлица из API Т-Банка, Сбера и Альфы, приводит их к единому формату и выгружает в Google таблицы. Пользователь видит банки, счета, период, список операций, статусы синхронизации, ошибки и дубли.

## Аналитика кейса

### Доступ к данным:

У всех трёх банков есть способ получить выписки и транзакции по счетам юрлиц

(https://developer.tbank.ru/docs/api/get-api-v-1-statement)

(https://sberbusinessapi-documentation.github.io/#2-5)

(https://developers.alfabank.ru/products/alfa-api/documentation/articles/transactions/articles/statement/v1/statement)
### Авторизация:

Т-Банк использует Bearer API Token (https://developer.tbank.ru/docs/intro/manuals/self-service-auth)

Сбербанк использует протокол авторизации OAuth 2.0 (https://sberbusinessapi-documentation.github.io/#2f1b86b685)

Альфа-Банк использует Bearer API Token и ApiKey (https://developers.alfabank.ru/products/alfa-api/documentation/articles/transactions/articles/statement/v1/statement)
### Формат данных:

Дата операции, дата проводки, сумма, валюта, тип операции (пополнение/списание), назначение, контрагент, ИНН контрагента, БИК Банка, ID транзакциии, статус.
### Что можно юрлицам в этих банках:

- Получать выписки и список операций по счетам компании;
- автоматически забирать транзации по API;
- передавать платёжные поручения;
- получать статусы оплат;
- интегрировать банк с учетной системой и таблицами

Что и подходит для этого кейса.

### Что можно физлицам:

- пользоваться ЛК или мобильным банком
- смотреть историю операций внутри банка
- иногда получать ограниченный open banking-доступ по согласию, если банк это отдельно поддерживает. (https://trends.rbc.ru/trends/industry/62bd42929a7947656ffa44d3)


## Как реализован прототип в песочнице:
## Режимы работы

### Демо-режим (MODE = "demo")

Не требует ключей банков. Скрипт сам генерирует тестовые транзакции с реалистичными данными: даты, суммы, контрагенты, назначения платежей. Нужен для демонстрации заказчику и отладки записи в Google Sheets.

Проверка запросов к банкам:
- Т-Банк: отправляется реальный запрос в песочницу с токеном `TBankSandboxToken`. При статусе 200 формат запроса верный
- Альфа-Банк: попытка подключения к песочнице, при SSL-ошибке используются тестовые данные
- Сбер: только тестовые данные (нет публичной песочницы)

### Production-режим (MODE = "production")

Требует реальные ключи от банков. Скрипт подключается к боевому API и получает настоящие операции по счёту.

## Как работает фоновый режим

Скрипт запускается один раз и работает постоянно. Планировщик APScheduler каждые N минут (настраивается) запускает синхронизацию. Скрипт можно остановить Ctrl+C.

## Как избегает дублей

Есть локальная база данных SQLite. В ней хранятся ID всех уже обработанных транзакций. При каждом запуске скрипт получает свежие операции от банка, сверяет ID с БД и добавляет только те, которых там нет.

## Как пишет в Google Sheets

При первом запуске создаётся лист с заголовками. При последующих запусках скрипт определяет последнюю заполненную строку и добавляет новые данные в конец. Старые строки не удаляются и не перезаписываются.

## Настройка

1. Установить библиотеки:
```bash
pip install google-auth google-api-python-client requests apscheduler
```

2. Настроить Google Sheets:
- Создать сервисный аккаунт в Google Cloud Console
- Включить Google Sheets API
- Скачать JSON-ключ, переименовать в `creds.json`
- Создать таблицу, добавить email сервисного аккаунта в редакторы
- Вставить ID таблицы в переменную `SHEET_ID`

3. Выбрать режим и банки в настройках скрипта:
```python
MODE = "demo"                      # demo или production
BANKS = ["tbank", "alfabank", "sber"]  # какие банки синхронизировать
SYNC_INTERVAL_MINUTES = 60         # как часто синхронизировать
SHEET_ID = "YOUR_SHEET_ID"         # ID из URL таблицы
WORKSHEET_NAME = "Bank_Transactions"  # имя листа
```

4. Запустить:
```bash
pip install google-auth google-api-python-client requests apscheduler
python bank_to_google.py
```

## Что получается на выходе

В Google Sheets создаётся лист с колонками:
- Банк, ID операции, Дата, Сумма, Валюта
- Тип (пополнение/списание), Назначение платежа
- Контрагент, ИНН, БИК, Статус
- Время синхронизации

Каждый запуск добавляет новые строки в конец.

## Переход в production

1. Получить ключи в банке:
   - Т-Банк: Client ID и API ключ в кабинете Т-Бизнес
   - Альфа-Банк: OAuth2 токен в Alfa Developer Hub
   - Сбер: OAuth2 токен и сертификат

2. Заполнить переменные окружения или вписать ключи в скрипт

3. Сменить `MODE = "production"`

4. Выбрать нужные банки `BANKS = ["tbank", "alfabank", "sber"]` (удалить лишние)

5. Определить интервал обновления `SYNC_INTERVAL_MINUTES = 0.2`
