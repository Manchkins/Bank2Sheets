# Bank2Sheets — Сбор банковских операций в Google Sheets.
Система автоматически собирает транзакции юрлица из API Т-Банка, Сбера и Альфы, приводит их к единому формату и выгружает в Google таблицы. Пользователь видит банки, счета, период, список операций, статусы синхронизации, ошибки и дубли.

## Аналитика кейса

### Доступ к данным:

У всех трёх банков есть способ получить выписки и транзакции по счетам юрлиц

(https://developer.tbank.ru/docs/api/get-api-v-1-statement)

(https://sberbusinessapi-documentation.github.io/#2-5)

(https://developers.alfabank.ru/products/alfa-api/documentation/articles/transactions/articles/statement/v1/statement)
### Авторизация:

Т-Банк использует Bearer API Token (https://developer.tbank.ru/docs/intro/manuals/self-service-auth)

Сбербанк использует протокол авторизации OAuth 2.0 (https://sberbusinessapi-documentation.github.io/#2f1b86b685)

Альфа-Банк использует Bearer API Token и ApiKey (https://developers.alfabank.ru/products/alfa-api/documentation/articles/transactions/articles/statement/v1/statement)
### Формат данных:

Дата операции, дата проводки, сумма, валюта, тип операции (пополнение/списание), назначение, контрагент, ИНН контрагента, БИК Банка, ID транзакциии, статус.
Что можно юрлицам в этих банках:

- Получать выписки и список операций по счетам компании;
- автоматически забирать транзации по API;
- передавать платёжные поручения;
- получать статусы оплат;
- интегрировать банк с учетной системой и таблицами

Что и подходит для этого кейса.

## Как реализован прототип в песочнице:

### 1. Получение данных от банков

Т-Банк: используется песочница с тестовым токеном TBankSandboxToken. Запрос к https://business.tbank.ru/openapi/sandbox/api/v1/statement с параметрами (номер счёта 20 цифр, даты). При статусе 200 запрос корректен.

Альфа-Банк: асинхронный API. Сначала POST-запрос на создание выписки, получение report_id, затем polling GET-запросов до статуса FINISHED. В песочнице SSL-проблемы, поэтому при ошибке — мок-данные.

Сбер и ВТБ: API изучены по документации, требуют OAuth2 и сертификаты. Публичных песочниц нет.

### 2. Инкрементальная загрузка

SQLite база данных хранит ID обработанных транзакций. При каждом запуске скрипт получает все операции за период, фильтрует новые (которых нет в БД), сохраняет их и отмечает как обработанные. Повторный запуск не дублирует данные.

### 3. Запись в Google Sheets

Сервисный аккаунт Google Cloud аутентифицируется через JSON-ключ (creds.json). Скрипт создаёт новый лист с датой в имени, очищает его, записывает заголовки и строки транзакций через batchUpdate.

### 4. Два режима работы

DEMO: генерирует мок-транзакции (случайные даты, суммы, контрагенты), не требует ключей банков. PRODUCTION: подключается к реальным API, требует заполнения ключей в настройках.

### 5. Структура выходных данных

Google Sheets содержит колонки: банк, ID операции, дата, сумма, валюта, тип (пополнение/списание), назначение платежа, контрагент, ИНН, БИК, статус, время синхронизации.

## Как будем реализовывать кейс:



## Быстрый старт:
```
pip install google-auth google-api-python-client requests
python bank_to_google.py
```
Перед запуском: создать сервисный аккаунт в Google Cloud, включить Sheets API, скачать JSON-ключ как creds.json, добавить email сервисного аккаунта в редакторы таблицы, указать SHEET_ID в скрипте.
